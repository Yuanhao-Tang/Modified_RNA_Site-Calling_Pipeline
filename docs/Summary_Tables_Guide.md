# Summary Tables Guide

This document describes the two main summary tables generated by the RNA modification pipeline.

## `sample_summary.csv`

### Purpose

This table summarizes one row per sample per stage and combines alignment, UMI, and coverage metrics.

### Typical Columns

- `sample`
- `stage`
- `aligner`
- `total_reads`
- `unique_aligned`
- `multi_aligned`
- `unaligned`
- `alignment_rate`
- `umi_input_reads`
- `umi_output_reads`
- `umi_dedup_rate`
- `coverage_total_reads`
- `coverage_mean_read_length`
- `coverage_mean_coverage`

### Suggested Interpretation

- Use alignment metrics to assess mapping quality.
- Use UMI metrics to evaluate deduplication behavior.
- Use coverage metrics to estimate sequencing depth and final usable signal.

## `coverage_summary.csv`

### Purpose

This table summarizes chromosome-level coverage for each sample and each stage.

### Typical Columns

- `sample`
- `stage`
- `chromosome`
- `covered_bases`
- `mean_coverage`
- `total_bases`

### Suggested Interpretation

- Inspect chromosome-specific depth variation.
- Check for unusually high mitochondrial or hotspot-like coverage.
- Compare the same chromosome across samples for consistency.

## Relationship Between The Two Tables

- `sample_summary.csv` is stage-level and compact.
- `coverage_summary.csv` is chromosome-level and more detailed.
- Together they provide both a global and a chromosome-resolved view of alignment quality and coverage.
